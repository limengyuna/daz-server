# 活动支出 & 分摊接口文档

> 基础路径：`/api`
>
> 需要登录的接口在请求头中携带：`Authorization: Bearer {token}`

---

## 1. 添加一笔支出

**POST** `/api/activities/{activityId}/expenses`

**需要登录**：是（从 Token 中获取付款人ID）

### 请求头

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
Content-Type: application/json
```

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | String | ✅ | 支出标题（如：午餐、门票） |
| amount | Number | ✅ | 总金额（元），必须大于0 |
| category | Integer | ❌ | 消费分类：0-其他, 1-餐饮, 2-交通, 3-住宿, 4-门票, 5-购物，默认0 |
| images | String[] | ❌ | 凭证/小票图片URL数组 |
| remark | String | ❌ | 备注 |
| splitType | Integer | ❌ | 分摊方式：1-均摊（默认）, 2-按比例, 3-指定金额 |
| splitUserIds | Long[] | ❌ | 参与分摊的用户ID列表，为空时默认所有已通过的参与者 |
| customAmounts | Object | ❌ | splitType=3时必填，格式：`{ "用户ID": 金额 }` |

### 请求示例（均摊）

```json
{
  "title": "午餐火锅",
  "amount": 200.00,
  "category": 1,
  "images": ["https://example.com/receipt1.jpg"],
  "remark": "海底捞",
  "splitType": 1
}
```

### 请求示例（指定金额）

```json
{
  "title": "打车费",
  "amount": 50.00,
  "category": 2,
  "splitType": 3,
  "splitUserIds": [1, 2, 3],
  "customAmounts": {
    "1": 20.00,
    "2": 20.00,
    "3": 10.00
  }
}
```

## 4. 删除某笔支出

**DELETE** `/api/activities/{activityId}/expenses/{expenseId}`

**需要登录**：是（只有付款人可以删除）

### 请求头

```
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

### 请求示例

```
DELETE /api/activities/5/expenses/1
```

### 返回示例（成功）

```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

### 返回示例（失败 - 不是付款人）

```json
{
  "code": 500,
  "message": "只有付款人可以删除该支出",
  "data": null
}
```

## 5. 获取活动账单汇总

**GET** `/api/activities/{activityId}/settlement`

**需要登录**：否

**说明**：计算该活动中所有支出的汇总，包括每个人的收支情况和简化后的债务关系。

### 请求示例

```
GET /api/activities/5/settlement
```

### 返回示例

**场景说明：**
- 小明（userId=1）垫付了 200 元午餐 + 60 元门票 = 共垫付 260 元
- 小红（userId=2）垫付了 90 元打车费
- 3人均摊所有支出，每人应承担 (200+90+60)/3 = 116.67 元

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "activityId": 5,
    "totalAmount": 350.00,
    "expenseCount": 3,
    "debts": [
      {
        "debtorId": 3,
        "debtorNickname": "小李",
        "debtorAvatar": "https://example.com/avatar3.jpg",
        "creditorId": 1,
        "creditorNickname": "小明",
        "creditorAvatar": "https://example.com/avatar1.jpg",
        "amount": 116.67,
        "settled": false
      },
      {
        "debtorId": 2,
        "debtorNickname": "小红",
        "debtorAvatar": "https://example.com/avatar2.jpg",
        "creditorId": 1,
        "creditorNickname": "小明",
        "creditorAvatar": "https://example.com/avatar1.jpg",
        "amount": 26.66,
        "settled": false
      }
    ],
    "personalSummaries": [
      {
        "userId": 1,
        "nickname": "小明",
        "avatarUrl": "https://example.com/avatar1.jpg",
        "totalPaid": 260.00,
        "totalOwed": 116.67,
        "balance": 143.33
      },
      {
        "userId": 2,
        "nickname": "小红",
        "avatarUrl": "https://example.com/avatar2.jpg",
        "totalPaid": 90.00,
        "totalOwed": 116.67,
        "balance": -26.67
      },
      {
        "userId": 3,
        "nickname": "小李",
        "avatarUrl": "https://example.com/avatar3.jpg",
        "totalPaid": 0.00,
        "totalOwed": 116.66,
        "balance": -116.66
      }
    ]
  }
}
```

**字段说明：**
- `debts` — 简化后的债务关系，表示"谁应该给谁转多少钱"
- `personalSummaries` — 每个人的收支汇总
  - `totalPaid` — 该用户一共垫付了多少钱
  - `totalOwed` — 该用户一共应该承担多少钱
  - `balance` — 净额，正数=别人欠我，负数=我欠别人

## 6. 标记分摊为已结清

**PUT** `/api/expenses/{expenseId}/splits/{splitId}/settle`

**需要登录**：否（建议前端自行控制权限）

### 请求示例

```
PUT /api/expenses/1/splits/2/settle
```

### 返回示例（成功）

```json
{
  "code": 200,
  "message": "已标记为结清",
  "data": null
}
```

### 返回示例（失败 - 已结清或不存在）

```json
{
  "code": 500,
  "message": "结清失败，记录不存在或已结清",
  "data": null
}
```

## 附录：枚举值说明

### 消费分类 category

| 值 | 说明 |
|----|------|
| 0 | 其他 |
| 1 | 餐饮 |
| 2 | 交通 |
| 3 | 住宿 |
| 4 | 门票 |
| 5 | 购物 |

### 分摊方式 splitType

| 值 | 说明 |
|----|------|
| 1 | 均摊（默认，总金额平均分给所有参与者） |
| 2 | 按比例（暂未实现） |
| 3 | 指定金额（需要传 customAmounts 字段） |

### 结清状态 isSettled

| 值 | 说明 |
|----|------|
| 0 | 未结清 |
| 1 | 已结清 |
